## **Requisitos Não Funcionais (RNF)**

São restrições que definem a qualidade do sistema — como desempenho, usabilidade e confiabilidade. Não tratam das funções, mas influenciam diretamente na arquitetura e seguem padrões de qualidade.

## **Diagramas**

→ Casos de Uso: Interação de Usuário e Administrador com o sistema.

→ Atividade: Fluxo do processo de login.

→ Classes: Estrutura de classes e seus relacionamentos.

## **Programação Orientada a Objetos (POO)**

→ Encapsulamento: Esconde os detalhes de implementação e expõe apenas a interface necessária.

→ Herança: Permite que uma classe herde atributos e métodos de outra.

→ Polimorfismo: Permite que um método se comporte de forma diferente com base no objeto que o chama.

→ Abstração: Foca nas características essenciais do objeto, escondendo os detalhes irrelevantes.
